## 1. 제품 비전 및 문제 정의

### 1.1 해결하려는 핵심 문제

Long-running AI 에이전트를 밤새 돌리려는 빌더들의 최대 공포는 "잘 될 때"가 아니라 **"망가질 때"**이다.

**실제 시나리오:**
- Claude Max 요금제를 끊어놓고 에이전트를 병렬 실행했지만, **권한 오류로 10분 만에 전부 실패**
- Google Sheets API 토큰이 자정에 만료되어 크롤링 500건 중 50건에서 멈춤
- Notion MCP 서버가 점검 중이어서 최종 결과 저장이 전부 실패
- 랩탑이 슬립 모드에 진입하여 tmux 세션이 죽음
- Python 가상환경이 시스템 업데이트로 깨짐

### 1.2 제품 한 줄 정의

> 사용자가 에이전트에게 맡길 작업을 설명하면, 그 작업이 성공적으로 완료되기 위한 **환경 전제 조건을 자동으로 도출**하고, **실제로 검증**하고, **자동으로 고칠 수 있는 건 고치고**, **실시간 웹 리포트**로 결과를 보여주는 Pre-flight Check 시스템

### 1.3 핵심 차별점

- **웹 대시보드**: 검증 완료 후 로컬 브라우저에서 예쁜 리포트 자동 오픈
- **Claude Code 플러그인**: 어떤 프로젝트에서든 `/chaos-lab` 스킬로 실행
- **Auto-fix Self-healing**: 자동 수정 가능한 CRITICAL 항목을 즉시 수정 + 재검증
- **API 키 불필요**: 로컬 키워드 매칭 + 규칙 기반으로 완전 오프라인 동작
- **맥락 인식 체크리스트**: Seed의 `relevant_to` 태그를 기반으로 작업에 무관한 항목을 자동 제외하여 오탐(false positive) 최소화
- **액셔너블 리포트**: 단순 PASS/FAIL이 아닌, 각 실패 항목에 "왜 필요한지" + "구체적 해결 방법(명령어)"을 함께 제시

## 2. 핵심 워크플로우 (5단계 파이프라인)

```
┌──────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  Step 1: 요구사항 분석 (사용자 선택)                                     │
│  ┌─────────────┐     ┌──────────────────────────────────────────┐  │
│  │ 사용자 프롬프트│ ──→ │ Quick: 로컬 키워드 매칭 (즉시, 무료)      │  │
│  │ (자연어)     │     │ Deep:  Ouroboros 소크라테스 문답 (정밀)    │ ──→ Seed  │
│  └─────────────┘     └──────────────────────────────────────────┘  │
│                                                       │              │
│  Step 2: 체크리스트 자동 생성 (Layer 2 커스텀이 메인)          ▼              │
│  ┌────────────────────────────────────────────────────────────────┐  │
│  │  Seed 분석 → Layer 2(커스텀) 동적 생성이 핵심                    │  │
│  │  Layer 1(베이스라인)은 참조용 안전망, Layer 2가 메인 체크리스트   │  │
│  └───────────────────────────────┬────────────────────────────────┘  │
│                                  │                                   │
│  Step 3: Shell 기반 병렬 검증      ▼                                   │
│  ┌────────────────────────────────────────────────────────────────┐  │
│  │  각 항목을 Shell 명령어로 실행 → PASSED/FAILED/WARNING/SKIPPED   │  │
│  └───────────────────────────────┬────────────────────────────────┘  │
│                                  │                                   │
│  Step 4+5: Auto-fix + 리포트 (병렬)  ▼                                │
│  ┌────────────────────────────────────────────────────────────────┐  │
│  │  ┌─ 리포트 뼈대 생성 (PASSED/WARNING/SKIPPED 즉시 반영)        │  │
│  │  └─ Auto-fix 병렬 실행 (CRITICAL만)                            │  │
│  │       → 완료될 때마다 FAILED→HEALED 반영                       │  │
│  │       → 전체 완료 후 Verdict 재계산 + 최종 리포트 확정           │  │
│  │                                                                │  │
│  │  Pre-flight Report (웹 + Markdown + JSON)                      │  │
│  │  ├─ 🔴 CRITICAL (즉시 조치 필요)                                │  │
│  │  ├─ 🟡 WARNING (권장 조치)                                      │  │
│  │  ├─ 🟢 PASSED (검증 통과)                                       │  │
│  │  ├─ ⚪ SKIPPED (수동 확인 필요)                                  │  │
│  │  └─ 🔧 HEALED (Auto-fix로 자동 수정됨)                          │  │
│  └────────────────────────────────────────────────────────────────┘  │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```
